# Task 9: Watchdog Contract Alignment Evidence (CORRECTED)

## Why Intent-Only Approach Failed
Task spec explicitly stated: "Do NOT fake reminder delivery with state-only confirmation."
Previous approach downgraded actual delivery to intent logging, violating requirement.

## Root Cause Analysis
- README.md line 4: "watchdog有session_send的权限" (watchdog has session_send permission)
- ADR-0003 line 27: "Send session messages | Use session_send to notify original session"
- TOOLS.md line 180: Listed session_send as "Not available in toolset"
- Contradiction: Design docs say watchdog HAS delivery capability, TOOLS.md said unavailable

## Actual Delivery Mechanism
- SessionSender protocol in notifier.py (lines 43-56)
- send_to_session(session_id, message) method
- MockSessionSender in tests proves delivery works
- Not OpenClaw native session_send tool, but repo-owned delivery protocol

## Resolution
- AGENTS.md: Documents "Reminder Delivery via SessionSender protocol"
- TOOLS.md: Removed session_send from prohibited list, added note about SessionSender protocol
- Both now agree: delivery happens via SessionSender, not direct tool call
- Maintains narrow authority: delivery is logged and tracked
